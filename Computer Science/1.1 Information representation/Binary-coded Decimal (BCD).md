**Binary-coded Decimal** uses 4 digit binary to represent each denary number
	- 0000 = 0
	- 0001 = 1
	- 0010 = 2
	- 0011 = 3
	- 0100 = 4
	- 0101 = 5
	- 0110 = 6
	- 0111 = 7
	- 1000 = 8
	- 1001 = 9

Can be stored either as half a byte (4-bits) or one byte (2 4-bit numbers)

**One byte**

|0000|0011|3|
|-|-|-|
|0000|0010|2|

**Half a byte**

|0011|3|
|-|-|
|0010|2|
